Integração com Telegram via .env e putenv()

Objetivo

Explicar como o projeto genesisMultiLog carrega variáveis de ambiente do arquivo .env utilizando a função putenv(), sem dependência de bibliotecas externas.

Estrutura do arquivo .env

TELEGRAM_BOT_TOKEN=SEU_TOKEN_AQUI
TELEGRAM_CHAT_ID=SEU_CHAT_ID_AQUI

Script de Carregamento

O script bootstrap/load_env.php lê o arquivo .env e injeta as variáveis no ambiente usando putenv().

require_once __DIR__ . '/bootstrap/load_env.php';

$token = getenv('TELEGRAM_BOT_TOKEN');
$chatId = getenv('TELEGRAM_CHAT_ID');

Integração com o Logger

A classe Logging\TelegramHandler acessa diretamente getenv('TELEGRAM_BOT_TOKEN') e getenv('TELEGRAM_CHAT_ID') para enviar logs de erro automaticamente para o Telegram.

Benefícios

  • Sem dependência de bibliotecas externas
  • Simples de configurar
  • Seguro, desde que o arquivo .env não seja exposto publicamente
